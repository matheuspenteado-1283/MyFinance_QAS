# Skill: Processo de Criação e Elaboração de Software

**Objetivo:** Acelerar processos através da ideação, especificação, e desenvolvimento sistemático de soluções de software.

---

## Fase 1: Brainstorming e Ideação 🧠
**Objetivo:** Entender o problema, o processo atual e gerar ideias focadas em aceleração e eficiência.
- [ ] **Mapeamento do Problema:** Qual processo manual ou demorado precisa ser acelerado?
- [ ] **Identificação de Gargalos:** Onde as pessoas perdem mais tempo hoje?
- [ ] **Definição de Sucesso:** Como saberemos que o software deu certo? (Ex: "Reduzir o tempo de fechamento de 3 horas para 10 minutos").
- [ ] **Ideação Livre:** Que tipo de automação ou interface resolveria isso?

## Fase 2: Especificação (Tabelas, Telas e Funcionalidades) 📋
**Objetivo:** Transformar as ideias abstratas em um escopo técnico e visual claro.
- **Tabelas e Dados (Banco de Dados):**
  - [ ] Quais são as entidades principais? (Ex: Clientes, Despesas, Relatórios).
  - [ ] Quais informações cada entidade guarda? (Atributos e tipos de dados).
  - [ ] Como essas entidades se relacionam? (Ex: Um Cliente tem Várias Despesas).
- **Telas (Interface / UI):**
  - [ ] Listar todas as visões necessárias (Ex: Dashboard, Formulário de Entrada, Lista de Registros).
  - [ ] Definir o fluxo: Onde o usuário entra primeiro? Para onde ele vai depois?
- **Funcionalidades (Regras de Negócio):**
  - [ ] Definir ações principais (CRUD: Criar, Ler, Atualizar, Deletar).
  - [ ] Estabelecer regras de validação (Ex: "O valor não pode ser negativo").

## Fase 3: Desenvolvimento 💻
**Objetivo:** Construir a fundação e a aplicação seguindo a especificação.
- [ ] **Setup Inicial:** Configuração do ambiente, bibliotecas e estrutura do projeto.
- [ ] **Banco de Dados:** Criação das tabelas e esquemas definidos na Fase 2.
- [ ] **Back-end:** Desenvolvimento da lógica de negócios, cálculos e rotas/APIs.
- [ ] **Front-end:** Criação das interfaces, botões e formulários com foco em usabilidade.
- [ ] **Integração:** Conectar as telas (Front) com o banco de dados e a lógica (Back).

## Fase 4: Ajustes e Validação 🛠️
**Objetivo:** Testar a solução na prática e lapidar a experiência do usuário.
- [ ] **Testes Funcionais:** Simular os passos que o usuário fará no dia a dia.
- [ ] **Caça a Bugs:** Identificar erros, quebras de layout ou falhas na gravação de dados.
- [ ] **Refinamento de UI/UX:** A interface está intuitiva? O usuário precisa de muitos cliques para uma tarefa simples?
- [ ] **Implementação das Correções:** Ajustar o código com base nos testes realizados.

## Fase 5: Revisão e Entrega ✅
**Objetivo:** Garantir a qualidade final antes que o software seja oficialmente utilizado.
- [ ] **Validação do Objetivo:** O software realmente acelera o processo definido na Fase 1?
- [ ] **Revisão de Código (Opcional, mas recomendado):** O código está limpo, organizado e comentado?
- [ ] **Preparação do Ambiente:** Configurar a hospedagem ou o executável para uso real.
- [ ] **Entrega Final:** Disponibilizar o software para os usuários finais.

---

### 🤖 Como usar esta Skill com a Inteligência Artificial

Sempre que quiser criar um novo projeto ou acelerar uma tarefa, copie este arquivo (ou refira-se a ele) e inicie a conversa dizendo:

> *"Quero criar um software para acelerar meu processo de [NOME DO PROCESSO]. Vamos usar a nossa **Skill de Criação de Software**. Por favor, inicie fazendo as perguntas da **Fase 1: Brainstorming**, uma de cada vez, para mapearmos a solução."*
